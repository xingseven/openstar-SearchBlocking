# Changelog

## 1.0.6 - 2026-05-27

- 修复候选结果去重逻辑，改为优先保留最内层卡片，避免把整块结果容器误判后整页隐藏。
- 新增根容器保护（如 `#b_content/#b_results` 等），禁止主容器进入屏蔽候选。
- 收紧通用扫描父容器回溯策略，避免右侧卡片修复引入“全页空白”副作用。

## 1.0.5 - 2026-05-27

- 修复 Bing 右侧/侧栏小卡片（`#b_context`、`#b_dynRail`）无法屏蔽的问题。
- 通用扫描不再跳过 `bing.com/ck/a` 跳转链接，并提升父容器识别深度。
- 新增 Bing 跳转链接解码匹配，会按真实目标地址（如 `blog.csdn.net`）参与关键词命中。

## 1.0.4 - 2026-05-27

- 内容脚本开启 `all_frames` 注入，覆盖同域 iframe 场景，避免“主页面已注入但结果区未命中”。

## 1.0.3 - 2026-05-27

- 扩大 Bing 结果扫描范围，新增 `#b_topw`、`#b_content` 等区域，覆盖顶部大卡片结果。
- 匹配逻辑升级为“正文文本 + 链接地址 + 紧凑匹配”联合命中，提升对 `CSDN` 类关键词的识别稳定性。
- 弹窗状态新增“已隐藏 X/Y 条结果”实时反馈，便于判断是注入问题还是命中问题。

## 1.0.2 - 2026-05-27

- 调整内容脚本注入策略为 `<all_urls>`，由脚本内再判断搜索引擎页面，避免地区域名遗漏导致不生效。
- 新增弹窗注入状态检测，直接提示“已注入/未注入/非搜索页”，方便快速排查。
- 增加 `sb-ping` 消息通道，用于确认当前标签页是否已加载内容脚本。

## 1.0.1 - 2026-05-27

- 扩展搜索引擎匹配范围，支持更多地区子域名，例如 `cn.bing.com`、`google.com.hk`。
- 重写搜索结果识别逻辑，优先定位结果卡片容器，匹配失败时自动回退到通用扫描。
- 修复关键词变更后结果不会重新显示的问题，支持实时恢复和重新屏蔽。
- 优化弹窗关键词去重逻辑，避免仅大小写不同的重复项。

## 1.0.0 - 2026-05-27

- 初始版本，支持在常见搜索引擎页面按关键词隐藏搜索结果。
